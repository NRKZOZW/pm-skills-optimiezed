# pm-journey-cpf

Customer Problem Fit (CPF) journey plugin: validate that customers truly have a problem worth solving. Guides PMs through market research, customer discovery, persona creation, and problem validation.

## Skills (4)

- **problem-statement** — Create a testable problem hypothesis using a structured template with target customer, problem, context, negative outcome, workaround, and limitation.
- **customer-discovery-plan** — Plan a customer discovery sprint: who to talk to, how to recruit, what to ask (following The Mom Test principles), and how to synthesize findings.
- **problem-validation** — Score a problem hypothesis using the Problem Validation Canvas — five dimensions scored 1-5 with a GO/PIVOT/KILL recommendation.
- **empathy-map** — Create empathy maps with six quadrants (See, Hear, Think, Feel, Say, Do) centered on Goals and Pains, filled from interview data.

## Commands (2)

- `/pm-journey-cpf:run-cpf` — Run a full Customer Problem Fit validation cycle — from problem hypothesis through customer discovery to evidence-based GO/PIVOT/KILL decision.
- `/pm-journey-cpf:validate-problem` — Score a problem hypothesis against the Problem Validation Canvas — a quick CPF health check.

## Fit Journey Context

This is **Stage 1** of the Fit Journey (CPF → PSF → SPF → PMF → GTM).

**Inputs**: Product idea or problem hypothesis
**Outputs**: Validated problem statement, personas, empathy maps, customer insights, problem score, market size estimate
**Next Stage**: Run `/run-psf` to validate your solution (pm-journey-psf)
