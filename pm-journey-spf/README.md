# pm-journey-spf

Solution Product Fit (SPF) journey plugin: build and validate your MVP. Guides PMs through MVP scoping, requirements definition, KPI setting, sprint planning, and user feedback analysis to verify users find real value.

## Skills (3)

- **mvp-scoping** — Define MVP scope using user story mapping, riskiest assumption testing, and MoSCoW prioritization to draw the line between what must ship now and what can wait.
- **kpi-framework** — Set KPIs for MVP success using North Star Metrics, input metrics, and health guardrails with clear GO/ITERATE/PIVOT thresholds.
- **feedback-loop** — Build systematic user feedback collection and analysis processes, turning raw feedback into prioritized backlog items.

## Commands (2)

- `/pm-journey-spf:run-spf` — Run a full Solution Product Fit cycle from MVP scoping through development planning, launch, and user validation.
- `/pm-journey-spf:scope-mvp` — Define MVP scope with core features, priorities, and success criteria in a focused 10-minute exercise.

## Fit Journey Context

This is **Stage 3** of the Fit Journey (CPF → PSF → SPF → PMF → GTM).

**Inputs**: Validated solution concept, value proposition, prototype results (from PSF)
**Outputs**: Released MVP, KPI baselines, initial user feedback, retention data
**Previous Stage**: Run `/run-psf` to validate your solution (pm-journey-psf)
**Next Stage**: Run `/run-pmf` to measure and achieve product-market fit (pm-journey-pmf)
